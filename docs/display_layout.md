# 画面レイアウト仕様

## パラメータ名と数値の桁数対応

MDSDRV-mml の音色定義コメント行（`;AR5 DR5 SR5 RR4 SL4 TL7  K2 ML4 DT2 SSG`）には、
パラメータ名とビット幅が示されている。
画面表示では **パラメータ名の列幅と、その下に並ぶ数値の最大桁数を一致させる** こと。

---

## 各パラメータの最小値・最大値と十進表示桁数

YM2612 レジスタフィールドおよび MDSDRV-mml の定義に基づく値域を示す。
**十進最大桁数**とは、その範囲の整数をそのまま表示したときに必要になる最大の桁数（`0` は 1 桁と数える）。

### グローバルパラメータ（音色全体）

`@0 fm ALG FB` の末尾 2 値はアルゴリズム（ALG）とフィードバック（FB）であり、**編集対象のパラメータとして扱う**。
いずれも **3 ビット**（アルゴリズム・フィードバックレジスタ）で、**0〜7**。

| パラメータ名 | ビット幅 | 最小値 | 最大値 | 十進最大桁数 | 備考 |
|-------------|---------|--------|--------|-------------|------|
| ALG         | 3       | 0      | 7      | 1           | 最大値 7 は 1 桁 |
| FB          | 3       | 0      | 7      | 1           | 同上 |

### OP パラメータ一覧（OP1〜OP4 共通）

| パラメータ名 | ビット幅 | 最小値 | 最大値 | 十進最大桁数 | 備考 |
|-------------|---------|--------|--------|-------------|------|
| AR          | 5       | 0      | 31     | 2           | 最大値 31 |
| DR          | 5       | 0      | 31     | 2           | 同上 |
| SR          | 5       | 0      | 31     | 2           | 同上 |
| RR          | 4       | 0      | 15     | 2           | 最大値 15 |
| SL          | 4       | 0      | 15     | 2           | 同上 |
| TL          | 7       | 0      | 127    | 3           | 最大値 127 |
| K           | 2       | 0      | 3      | 1           | Key scale（KS） |
| ML          | 4       | 0      | 15     | 2           | Multiple |
| DT          | 3 ※     | 0      | 7      | 1           | Detune（YM2612 は DT1〜DT3 の 3 ビット） |
| SSG         | 4       | 0      | 15     | 2           | SSG-EG |

※ DT は MML コメントで `DT2` と書かれることがあるが、YM2612 上は **0〜7** の 3 ビットフィールドとして扱う（本プログラムもこれに合わせる）。

---

## レイアウト用の数値列幅（右詰め）

十進最大桁数に加え、**ラベル文字数**を満たす必要がある列は、列幅をラベルに合わせる。
数値は **右詰め** とし、列幅に満たない分は先頭をスペースで埋める。

| パラメータ | 十進最大桁数 | ラベル文字数 | レイアウト数値列幅（文字） | 根拠 |
|-----------|-------------|-------------|---------------------------|------|
| ALG       | 1           | 3（`ALG`）  | **3**                     | max(1, 3) = 3（ラベル列に合わせる） |
| FB        | 1           | 2（`FB`）   | **2**                     | max(1, 2) = 2 |
| AR        | 2           | 2           | **2**                     | 2 |
| DR        | 2           | 2           | **2**                     | 2 |
| SR        | 2           | 2           | **2**                     | 2 |
| RR        | 2           | 2           | **2**                     | 2 |
| SL        | 2           | 2           | **2**                     | 2 |
| TL        | 3           | 2           | **3**                     | 3（最大 127 の 3 桁が支配的） |
| K         | 1           | 1           | **1**                     | 1 |
| ML        | 2           | 2           | **2**                     | 2 |
| DT        | 1           | 2           | **1**                     | 十進最大桁数 1。列間に最低 1 タイルの隙間を確保するため `TL K ML` 側の配置で調整 |
| SSG       | 2           | 3（`SSG`）  | **3**                     | max(2, 3) = 3（ラベル列に合わせる） |

上記 **レイアウト数値列幅** は `inc/fm_display.h` の `FM_VAL_WIDTH_*` と一致させる（OP 列は `src/fm_display.c` の `OP_VAL_WIDTH[] 初期化` に同じ定数を並べている）。

### 留意事項

- **ラベル行と数値行の列位置は原則揃える。**
  ラベルが2文字のパラメータ（AR, DR, SR, RR, SL, ML）は2文字分の列幅を確保し、
  数値は右詰めまたはゼロ埋めで表示する。
- **例外（ML / DT）:** パラメータ名行だけ **`MLDT`** と連続表示してよい。
  **数値の OP 列位置は `ML` と `DT` を分けたレイアウト**（`OP_COL_X`）のままとし、ラベルの「MLDT」と横位置を一致させない。

- **TL のみ3桁** であるため、隣接するパラメータとの区切りスペースに注意すること。
  MMLコメント行でも `TL7` の前後にスペースが多めに入っているのはそのため。
  **SL と TL のラベル間**は **2 スペース**（`… SL  TL …`）。

- **K, DT は1桁** であり、ラベルも1〜2文字と短い。
  他のパラメータと並べる際、列が詰まりすぎないよう最低1文字の余白を設ける（数値行の ML・DT 間など）。

- カーソルはパラメータ名の真下（数値行と同じ列）に表示する。
  ML/DT 以外ではラベル幅と数値幅を一致させること。ML/DT は上記例外どおりラベル（MLDT）と数値列のズレを許容する。

### カーソルスプライト画像（`res/`）

- `cursor_1digit.png`（幅 8px）／`cursor_2digit.png`（16px）／`cursor_3digit.png`（24px）は、**互いに同一のインデックスカラーパレット**を共有する。
- そのパレットは **パラメータ数値テキストの描画に使っているパレット（`PAL0`）と CRAM 上で兼用できる**想定とする（スプライト用タイルとテキスト用色のインデックスを衝突させないよう設計する）。

- **操作:** D-Pad でセル間を移動する（実装は `fm_cursor.c`）。グローバル行（ALG / FB）と OP 行（OP1〜4 × AR…SSG）を上下で行き来する。左右は行内のセル移動。数値編集は別途。

### 表示イメージ（参考）

```
ALG FB
  3  0

AR DR SR RR SL  TL K MLDT SSG
31  0 19  5  0 23 0  0  0   0   ← OP1（数値は `ML` `DT` 分離レイアウト。ラベルだけ MLDT）
31  6  0  4  3 19 0  0  0   0   ← OP2
31 15  0  5  4 38 0  4  0   0   ← OP3
31 27  0 11  1  0 0  1  0   0   ← OP4
      ^                           ← カーソル例（SR列）
```

ALG/FB もカーソルで選択・変更できる。カーソルがALG/FB領域とOP領域を行き来できるよう設計すること。

数値は右詰め表示とし、ラベル幅に満たない場合は先頭をスペースで埋める。

## 確認用音程

### 概要

現在の発音音程を画面のどこかに常時表示する。
表示フォーマットは音名＋オクターブ（＋シャープ記号）で、例：`D4+`（D#4）。

### 音程の範囲

| 境界 | 音程 | 備考 |
|------|------|------|
| 最小 | o0 C | C0（最低音） |
| 最大 | o8 B | B8（最高音） |

最小・最大でクランプし、範囲外には移動しない。

### 操作

| 操作 | 動作 |
|------|------|
| B ＋ 左右 | 音程を **半音 ±1** 変更 |
| B ＋ 上下 | 音程を **±1オクターブ** 変更 |

### 表示フォーマット

```
NOTE: D4+
```

- 音名は `C` `C+` `D` `D+` `E` `F` `F+` `G` `G+` `A` `A+` `B` の12音
- `+` はシャープ（半音上）を示す
- オクターブ番号はMDSDRV表記（o0〜o8）に合わせる
- 表示幅は最大4文字（例: `B8` など）になるよう確保する
